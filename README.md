# 营造 yingzao · Skill 打磨工坊

> **把"能用"的 skill 打磨到"能交付"。** 你和同事写的 skill 自己能跑，但别人看不懂、不敢装、装了三分钟跑不通？营造用统一的九维标准量出差距，再用过验证门的改写把它修到位——PM、研发、设计、运营的 skill 都适用。

**双档**：
- **查勘（快检档，默认）**：10 分钟，全程只读，一页体检报告（九维分 + P0/P1 清单 + 一句话建议）
- **大修（精装档）**：七步全流程（立项审查 → 同类调研 → 生态定位 → 质量评测 → 差距规划 → 定向改写 → 验收交付），改写必须过验证门（实测优于原版才接受）

> 🏗️ **试点期**：当前 v1.0 出厂验证已通过；正在对首批 skill 做完整大修积累真实案例，效果示例区随试点更新。

## 快速开始

```bash
git clone <本仓库> && cd yingzao
./install.sh claude-code        # 安装到 ~/.claude/skills/（默认）
./install.sh cursor --project /path/to/project
./install.sh codex
./install.sh opencode --project /path/to/project
```

## 触发方式（对 Agent 说）

| 你说 | 触发 |
|------|------|
| 「查勘 prd-writer」「体检一下我这个 skill」 | 查勘（只读快检） |
| 「大修 xxx」「发布前完整打磨 xxx」「xxx 出师」 | 大修（七步全流程） |
| 「**比样**大修 xxx」「多候选打磨」「best-of-3」 | 大修 + 比样模式（每轮 3 候选 Pareto 取优，耗时约 ×2） |
| 「帮同事看看这个 skill」「这个 skill 能发布吗」 | 按语境进查勘或大修 |

## 三个可选输入（不给也能跑）

1. **岗位画像**（PM/研发/设计/运营）：决定对标搜索源和报告用语——给运营的报告不写工程黑话；
2. **团队内部源**：在 `yingzao/references/roles.md` 顶部登记团队 skill 仓库，访例优先对标自家同事；
3. **发布目标**（内部=默认/开源）：内部目标不强求 README 和 demo；声明开源时给出完整出门清单。

## 成本预期（设计期粗估，试点实测校准）

| 档位 | tokens | 适用 |
|------|--------|------|
| 查勘 | 30-60K | 日常自查、互检 |
| 大修·非隔离估分档 | 0.2-0.5M | 宿主无隔离能力时（分数标"估分"） |
| 大修·隔离实测档 | 0.5-1.5M | 完整 harness：裸基线/原版/改写版隔离运行 |
| 大修·比样档 | 1-3M | 高价值 skill 发布前 |

## 安全边界（营造不会做什么）

- **查勘绝对只读**；大修对目标的一切写入默认只产出候选 patch，**祈使句授权后才落盘**（疑问句只是问状态）；
- 测试运行默认 **dry-run**——被测 skill 的发布/删除/外呼动作一律先列计划等授权，不会替你执行真实副作用；
- **不执行 git commit / revert / reset**，版本操作权归你；非 git 目录自动快照备份，零 git 知识可回滚；
- 运行产物默认写 `~/.yingzao/runs/`（脱敏 ID 命名），**绝不默认写入你的项目**；
- 评分必须附证据；无实测总分上限 70；估分一律显式标注；对标全部带 URL，搜不到如实说"未访得"。

## 效果示例

> 出厂验证（2026-06-12）：对 PM 域方法论型 skill（多文件结构）与跨域工作流型 skill 各跑一轮查勘，报告见试点案例区（随试点期补充真实大修前后对比）。

## 文件结构

```
yingzao/
├── SKILL.md              # 主流程（双档七步 + 全部纪律）
├── references/
│   ├── scoring.md        # 九维评分细则 + 形态权重 + 规模适配
│   ├── roles.md          # 岗位画像 + 团队内部源 + 发布目标
│   ├── anti-patterns.md  # 禁例（15 条出厂 + 自迭代增量区）
│   └── case-log.md       # 打磨台账（public-safe）+ 岁修计数器
├── tools/inspect-skill.sh    # 规则预检（零 token，双段式，可挂 CI）
└── templates/                # 查勘/大修报告模板
```
