## 为何 T2/T3 当前只能估分（诚实标注）

按营造自己的禁例 17（裸基线须 skill 目录外 cwd + 禁读 skill 目录 + 剥离全局注入，否则子实例自己找到 skill 照执行=全污染），真实裸基线需要**硬隔离实例**。当前环境做不到硬隔离：
1. 主 Agent 自身已加载营造上下文，**禁例 17 明令不得自己假装裸 Agent**；
2. 起 subagent 仍能访问 skills 列表（含 yingzao），会「找到 skill 照执行」，不是真裸基线；
3. harness 层无法保证剥离 yingzao skill 访问 + 全局注入。

故 T2/T3 落**估分**（依据见上表），**不跑假裸基线冒充实测**——这正是营造「做不到硬隔离 → 估分、并标注」纪律的自我应用。

## 实测转化条件

当具备硬隔离实例（独立 cwd、禁读 yingzao 目录、剥 CLAUDE.md/memory/skills 注入）时，跑 T2/T3 裸基线，留 `baseline_output.md` / `with_skill_output.md`，把上表对应行从「估分」转「实测」。完整 6 类存档（bundle-manifest / isolation.txt / fs-before-after.sha256）作 stretch。

## 元收获

营造一边要求别人留审计存档、一边自己的 T2/T3 只有「高概率」一句预期——本轮补结构化估分 + 实测门槛 + 诚实标注「为何还没实测」。**和 npx 实测、跨-runtime 矩阵同源的「实测 > 纸面」自洽纪律**：做到哪标到哪，没做到的写清门槛，不拿估分冒充实测。
