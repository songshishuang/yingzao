# 结构化诊断 V/A/K（D5 · v1.10）

> 查勘/勘验对每个 P0/P1 缺陷强制产出结构化三元组 **D_i=(V_i, A_i, K_i)**，作大修前门控与 abstain 依据。
> 蒸馏自 SkillRevise(arXiv:2606.01139)§3.1——消融"诊断"使 SkillsBench 成功率 53/86→28/86（所有消融中影响最大），**诊断是承重件**。**硬门由 eval-harness 承担、本规范的契约脚本只校验结构存在、不校验语义。**

## 三字段

### V_i 验证规格（怎么算修好）
= 该缺陷专属的**四件套测试判据**（成功判据 / 失败判据 / 诱饵意图 / 不可接受输出黑名单，复用 before-after-protocol 评委判据定义）。
- **查勘版**：只写判据**草拟**（非承诺、不落测试文件）。
- **大修版**：必须实例化为可隔离重跑的测试 prompt，且**显式核对草拟与最终一致**（防脱节）。
- **双向兜底（V 无效不得进门）**：① 剔永真断言（裸基线也通过则重写）；② 须能区分「修好输出」与「已知坏输出诱饵」——无法区分坏样本的 V 判无效。

### A_i 失败归因（为何现在不行）
= failed checks + observed behavior + probable cause + defect label，映射九维①–⑨ + 四大高频病种（测试缺失 / 一致性漂移 / 真相源矛盾 / 死链）。
- **abstain 判据**：A_i 归因为「裸基线也犯 / 非 skill 级（底模问题）」→ 该项 **abstain、不进 P0**（呼应天花板 headroom：底模掩盖型的"缺陷"不是 skill 的锅）。

### K_i 保留约束（大修不得回退什么）
= 该 skill 当前**已通过、不得在大修中破坏**的检查/能力清单。两类：
- **语义不变类**（pass→fail 硬断流）：核心能力、安全红线、真实产物——大修后若这些从 pass 变 fail，**门0 断流、弃该候选**。
- **重构敏感类**（落架豁免）：措辞/结构等表层——落架重写时可变，不触发断流。

## 门0 · K 守恒（大修前置硬门）
大修候选先过门0：对 K_i 的「语义不变类」逐项复跑，**任一 pass→fail 即断流、弃候选**（落架模式豁免重构敏感类）。门0 在验证门①②③④之前。**弱宿主无隔离** → 门0 退化为软提示（不硬断流），报告标注。

## 与既有机制的映射
| V/A/K | 对接 |
|---|---|
| V → 测试 | before-after-protocol 四件套 + eval-harness 门① |
| A → 诊断 | 九维评分 + 四大高频病种 + abstain（headroom 底模掩盖型） |
| K → 守恒 | 门0 + 棘轮（分数只升不降）+ 落架豁免 |

## 报告承载
- **查勘**（quick-report）：精简三列「缺陷ID ｜ V 判据草拟 ｜ A 归因(九维ID+病种+是否skill级/abstain) ｜ K 不得回退」（只读、V 仅草拟）。
- **大修**（full-report §5b）：差距清单每 P0/P1 行含非空 V(可执行四件套/abstain豁免) / A(九维ID+病种+skill级?) / K(两类标注)；§7 加「门0 K守恒」列。
- 契约校验 `tools/validate-report-contract.sh`：校验大修报告 §5b 每 P0/P1 行 V/A/K 列**结构存在**（缺则断流），不校验语义。
