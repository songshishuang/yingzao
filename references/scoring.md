# 勘验 · 九维评分细则

> 营造自定义权重：制定时参考了 SkillLens（microsoft.github.io/SkillLens 项目页）维度有效性实证（失败模式 / 可执行具体性 / 高风险黑名单三维有实证支撑）与社区权重谱系，但数值是营造的启发式起点，**待 case-log 实测数据积累后在岁修中校准**。评分必须逐维附证据（引用目标文件的具体行/段），禁止凭印象打分。

## 硬规则

1. **估分标注**：未实测的维度一律标「估分」；估分维度占比 >30% → 报告显著告警。
2. **总分上限（缺一即触发，非满足其一即豁免）**：测试 prompt、前后对比、真实运行输出样例——**三者缺任意一项**即 → 总分 ≤70（写得再漂亮进不了优秀区）。**判例**：仅有 evals.json / 期望断言清单（expected_output）而**无真实运行输出存档、无 before/after 对比** → 仍属「缺真实输出样例」，**触发上限**——测试集「写了」不等于「跑了」（Y-011 dogfood 实测：此处措辞歧义曾致两评委对同一 skill 给分差 18）。
3. **四件套门槛**：测试 prompt 缺四件套（成功判据/失败判据/诱饵意图/不可接受输出）→ 不得进入第 9 维计分。
4. **盲评分离**：评分优先由独立子 Agent 执行；宿主不支持时用全新上下文评分 prompt 并标注「非严格盲评」。
5. **预测增益校准（v1.9）**：「预期提分 / 增益」一律给**保守区间**（下界默认 0），不给点估；显式区分**结构分 vs 真实任务输出分**——验证门只认输出分（结构分↑≠输出分↑）。原版属「底模掩盖型」（设想/实测裸基线已高、skill 本身 lift≈0）时，增益区间上界压低、优先建议维持/小修（详见 SKILL「画样 · headroom 预判」与 `references/before-after-protocol.md`）。**实证依据**：50 案随机社区样本，大修平均真实输出增益仅 +0.23、回归率 30%、自评预测 vs 实测偏置 +1.58（系统性高估）——故增益声明须保守、须用 before/after 实测而非文档观感。

## 形态判定特征与权重修正

多形态取主形态（按 SKILL.md 主体内容占比判断）。修正算法（v1.2 修订——旧版「从资源整合度/检查点设计等额扣减」在工作流型/风格型下产生负权重，不可执行）：

1. 该形态的侧重维度基准权重 ×1.25
2. **全表九维等比缩放归一到 100**：每维最终权重 = 调整后权重 × (100 / 调整后总和)，保留 1 位小数
3. 「实测表现」无论形态缩放后不得低于 20（实证核心维，低于则差额从最大维等比回collect）——通常缩放结果天然满足，此条为护栏

> 示例（工作流型）：检查点 6→7.5、失败模式 12→15，调整后总和 104.5，全表 ×(100/104.5)——九维全部微缩，无需指定扣减源，任何形态数学恒合法。

| 形态 | 判定特征 | 勘验侧重（×1.25 维度） |
|------|----------|------------------------|
| 工具型 | 包装脚本/CLI/API，主体是调用说明 | 失败模式覆盖、安全边界（脚本稳定性、依赖最小化、dry-run 能力） |
| 方法论型 | 编码一套分析/写作/决策框架 | 工作流清晰度、可执行具体性（输出模板质量、反例清单） |
| 工作流型 | 串联多步骤多工具 | 检查点设计（不扣此维）、失败模式覆盖（暂停点、断点恢复） |
| 风格型 | 文风/视觉/排版迁移 | 可执行具体性、实测表现（风格定义可被陌生 Agent 执行、before/after 对比强度） |

## 形态分流判据（v1.6 · 同一维度按形态认定判法，不改权重、不破历史口径）

> 上表解决「哪些维度更重要(×1.25)」；本节解决「同一维度对不同形态该怎么判」。**仅对「纯」形态触发**，复合 / 工作流 / 工具主形态按上表原判——故 case-log 历史五场（工作流 / 方法论复合 / 工具型为主）口径不破。Y-013 大修 dogfood 复评实证：对照组同形态 Δ ±3（评分噪声内）、改进组纯问诊型 +4~+8（仅第 8 维获益、仍受 ≤70 上限）。
>
> **纯形态判定特征**（消歧用，避免与复合形态混淆）：① 纯问诊 / 决策框架型 = 主体是 forcing-questions / 判定矩阵、提问驱动、产出诊断而非执行（如各 C-level review、合规 audit-prep）；② 纯认知框架型 = 零副作用 + 无资产脚本、主体是心智模型 / 分析框架陈述（如 altitude-horizon）；③ 纯创作型 = 文风 / 文案迁移、freeform 是正确特征（如 cold-email）。带资产脚本 / 多步编排 / 写副作用的归复合或工具型，不触发本节。

- **第 9 维 实测表现 · 证据类型按形态认定**：「实测」不限「Agent 行为测试 prompt」一种。无 Agent 行为测试 ≠ 实测 0，按形态认定等效真实证据——① 确定性脚本型：脚本 + **可复跑 golden-output**（expected_output 等对照文件，二者俱全）= 「脚本断言测试」，实测维给非零基线分（≤12）；仅有脚本 + sample 输入而无 golden 断言对照的不满足、仅给极低基线。② 纯问诊 / 决策框架型：改测「判定 / 分类正确性」（给定输入场景 → 是否正确识别到对应诊断 / 分支），附运行对照。**硬规则不松**：仍受「无任何真实运行对照 → 总分 ≤70」约束——本条只放宽「什么算真实运行对照」，不放宽「必须有对照」（佐证：a11y-audit 因带真实 expected_output 合理破 70）。
- **第 8 维 安全边界 · 判据按副作用面分流**：① 有副作用型（写文件 / 调 API / 执行命令 / 产出面向终端用户的诱导设计）：评「高风险动作黑名单 + 授权语义 + dry-run」（原判）；② 纯认知 / 分析型（零副作用的咨询 / 框架 / 打分）：「高风险黑名单」无适用对象，改评「决策边界（不替用户拍板）+ 认知边界（不造假数据 / 不过度解读统计）+ 领域合规提醒」是否清晰——三者俱全即可达满分，**不因「没有危险动作可禁」而扣分**。
- **第 4 维 检查点设计 · 创作型按交付门评**：① 流程 / 工作流型：评「关键节点有暂停 / 确认 / 自检 gate」（原判）；② 风格 / 创作型：freeform 是其正确特征（好文案不该被流程刚性束缚），改评「交付前是否有自检 / 质量门（read-aloud、quality-check 清单）」，**不因「中途无强制 gate」而扣分**。

## 规模适配（防误伤单文件 skill）

「资源整合度」「整体架构」按**组织合理性**评分，不按文件数量：内容在 500 行预算内的单文件 skill，单文件即最合理组织（不因「没有 references」扣分）；**超预算未拆分才扣**。单文件超预算的演进路径：查勘标超额 → 画样列「拆分 references」P1 项 → 细作执行。

## 评级

- **90-100 可出师**：可直接交付团队 / 申请开源出门清单
- **70-89 可用待精**：按 P1 清单打磨一轮
- **50-69 毛坯**：P0 清单优先，建议大修
- **<50 重新相地**：前提可能不成立，回到立项审查

## 已知盲点（岁修候选 · 暂不入权重，正式纳入须走大修流程）

九维当前是「单 skill 内部质量」视角；2026 学术与官方标尺已转向**生态与对抗**视角。以下维度尚未进入权重表（**避免破坏历史分数口径**），登记为岁修候选，当前作为评分时的**附加检查点**人工核查，正式纳入权重须走大修流程（配套 harness + 独立评委验证）：

1. **共存安全（Coexistence）**：单个 skill 单独安全，与其他 skill 同时加载时可能互相偷触发、或能力组合产生涌现风险（对标 Anthropic enterprise 5 维的 Coexistence；SkillReact, arXiv:2606.00448 实证约 1/5 配对级风险被逐 skill 扫描漏掉）。当前并入第 8 维安全边界**附加核查**，不单列计分——正式纳入须先具备多-skill 共载测试 harness。
2. **触发词对抗面**：第 1 维奖励「触发词丰富」是从**可用性**（被正确唤起）出发；但触发词同时是攻击面——可被构造以操纵语义检索、抢占其他 skill 的触发（arXiv:2605.11418，操纵成功率 86%）。评分时第 1 维须**两面看**：可用性面（触发词丰富、负触发明确 = 正分照给）+ 安全面（description 可被滥用操纵 → 计入第 8 维扣分），**不可只奖丰富**。
3. **供应链 / 依赖来源可信**：skill 引用的外部资源（references / scripts / MCP / 数据源 / 第三方仓库）来源是否可信、会否被投毒或替换——第 8 维当前查「死链 / 资源孤儿」但不查「依赖来源可信度」。**团队内部 skill 场景优先级低于共存**（内部源多可信），列附加核查、暂不入权重；当 skill 大量引用外部不可信来源、且 dogfood 实证发现系统性漏判时再走大修流程。（2026-06-14 补登记——06-13 竞品调研列「最该补」四盲点中唯一漏登的一条，收口归位；克隆检测经 luban 对比已论证为访例层冗余、不补。）
