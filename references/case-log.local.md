### YZ-DOGFOOD-001 · 2026-06-14 · 100-skill 大修模拟（准确性 × 可行性二次验证）

- **规模**：从 856 个真实 GitHub skill（deanpeters/Product-Manager-Skills · coreyhaines31/marketingskills · alirezarezvani/claude-skills，跨 13 领域）分层采样 100 个，模拟大修——全量诊断+画样 100 个 + 真改写盲评 20 个 + 独立复核 8 个。多 agent workflow 编排（68+4 agent / 347 万 token / 24 分钟）。
- **准确性证据**：复核平均 |Δ| **3.67 分**（5/6 ≤5）；**99/100 ≤70**（唯一破 70 的 a11y-audit=76 因带真实运行产物 expected_outputs，规则方向正确）；盲评诚实抓出 1 个改写被改坏 -16 分、16/20 过门（不偏袒）；预检死链 FAIL 经 v1.5.2 修复 **25→1**。
- **可行性边界**：① 批量子 agent **20% 跑偏**（没读输入 tsv、转去诊断本地 skill 甚至 yingzao 自己）→ 需 prompt 层硬约束输入边界 + 完成后 id 完整性校验（强约束补救 4/4 成功）；② 大修「实测验证门」**无法批量自动化**——细作提分均 +8.6 是「盲评提分」非「实测验证门提分」，后者必须逐个真跑（规模化天花板）。
- **改进点（形态分流补判据 · 待开发态走大修流程校准 scoring.md，不手改破棘轮）**：
  1. **实测维(23分)对"纯问诊/forcing-question 型"与"纯方法论框架型"偏苛**——价值在拷问/框架质量，无自带评测集即实测 0、压毛坯，低估实质成熟度（如 channel-economics 实达可用待精被压毛坯顶）。建议：脚本可复现/自带 golden-output 的给非零基线分。
  2. **安全边界(7分)对纯认知/分析型 skill 空转**——无副作用动作，"高风险黑名单"无适用对象。建议：按形态分流（行动型评黑名单+授权；认知型评决策/认知边界+领域合规）。
  3. **检查点设计对交互流/创作型 skill 偏苛**——freeform/即兴生成是形态特征非缺陷。
  → **与 [2026-06-13 20-skill 查勘] 结论完全一致，百级样本复现证明非偶发**，是九维最该补的真实改进点；属 scoring 核心改动，须走大修验证门，不手改。
