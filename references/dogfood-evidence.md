### YZ-REVIEW-002 · 2026-06-14 · SkillOpt 对比驱动（岁修候选：竞品启示 vs 实测信号优先级）

> 非 dogfood 批量验证，而是核实微软 SkillOpt（arXiv:2605.23904）真实机制后、与营造「访例→定式→画样」竞品驱动改进链对比产出的单条岁修候选。

- **背景**：SkillOpt 是纯执行轨迹驱动的**全自动**文本空间优化器——optimizer model 把 scored rollout 转成 bounded edits、只接受严格提升 held-out validation score 的改动，**完全不看同类竞品**，6benchmark×7model×3harness 52 配置全胜。对比发现：营造的**访例/定式（竞品参考）正是它偏离 SkillOpt 的地方**——SkillOpt 证明「不靠竞品也能优化得很好」，故营造竞品环节的正当性 = 服务 SkillOpt 跑不起来的场景（无可打分 benchmark / 生态位定位 / 非工程岗位）。
- **待观察缺口 · 外生对标牵引方向**：定式「对标强于我的维即候选差距」(SKILL.md 定式步) 是**外生信号**，可能诱导补齐"竞品有但目标不需要"的维、稀释独特定位；当前靠相地独特性锚定 + T1「不采纳」**人工判断**纠偏，非 SkillOpt 那种「validation score 客观裁决」。即营造用「无 benchmark 也能优化 + 生态位定位」换来了「优化方向依赖人工纠偏」的代价。
- **改进点（细作 T1 转化桥加判据）**：**当目标已有可跑的测试 prompt / golden-output 时，竞品启示与「实测低分项」冲突的，以实测为准**（内生分数优先于外生对标）。把「内生优先」写成明文纪律，补上「靠人工判断纠偏」软肋，且与细作自述「SkillOpt 式受控迭代」(SKILL.md 细作步) 对齐——SkillOpt 正是纯内生轨迹优先。
- **边界（防重蹈空转覆辙）**：仅在「有实测资产」时触发内生优先；无 benchmark 的团队场景（营造主场）竞品对标仍是**唯一可行替代信号**，不动。属形态/场景相关，非普适改权重。
- **触发条件（实证驱动，类比 v1.6 形态分流路径）**：当大修对象自带可跑测试资产、且 dogfood 实证发现访例对标差距与实测低分系统性冲突时，再走大修验证门把此判据补入细作 T1（不普适、不动权重、不破历史口径）。在此之前**只观察、不动核心**。
- **来源**：2026-06-14 SkillOpt（arXiv:2605.23904）真实机制核实（WebFetch + WebSearch 交叉验证）+ 营造竞品驱动链对比。

## YZ-POSBIAS-D1 · 盲评门位置偏置随机性验证（2026-06-16 · v1.10 Wave1 前置实验）

- **背景**：D1 拟加「换序双判」消位置偏置，但换序双判是为**跨模型判官**设计；yingzao 是**同族单评委**（Claude 评 Claude），若位置偏置**系统性偏一侧**，双判会把系统偏好转平局、毁信号。故先实验定性，再决定 M1 走向。
- **方法（预注册·跑前定阈值）**：取已有 50 案的 36 个 (case,题) 输出对，每对换序判两次（序 AB / 序 BA），测位置一致率 + 首位选择率 + 二项检验。预注册判据见 `reports/skill-study-2026-06-15/posbias/PREREG.md`。
- **结果（实测·n=36 对/72 次判定）**：位置一致率 **0.667**（换序约 **1/3 翻转**）；首位选择率 **0.389**（28/72，二项检验 p≈0.077 **不显著偏一侧**）。
- **判定：高噪型**（一致率低但首位无系统偏）→ **换序双判成立且价值最大**（专消随机翻转噪声，非系统偏，不会毁信号）。
- **回溯含义**：2026-06-15 那次 50 案评测仅做 A/B 随机、未换序双判，故其单案胜负带约 1/3 位置噪声——方向稳健、个案需打折看；这正是 D1 换序双判要修的。
- **落地**：`tools/eval-harness.sh` 加换序双判模式（条目带 `order`→同评委两序都胜才记胜·矛盾不计胜），3 夹具自测过（v2-compat / swap-consistent-win / swap-disagree）。
- **来源**：`reports/skill-study-2026-06-15/posbias/`（PREREG.md / results.json / verdict.json / analyze.py）；处方 MT-Bench(arXiv:2306.05685)。

## YZ-CALIB-D3 · 预测校准量化（2026-06-16 · v1.10 Wave2 D3）

- **机制**：eval-harness 第一层逐 run 点对账——scores.json 含 `pred_gain`（画样点估）时算 `calib_err = realized − pred_gain`（**负=本轮高估**），累计 `Bias = mean(calib_err)`。第二层（区间覆盖率/PICP 回调）预留未生效（真实门① run 现状=0、需 ≥5–8 run）。
- **符号统一**：v1.9 报"偏置 +1.58 系统性高估" = `pred − realized` = `−Bias`；文档/工具一律以 `realized − pred`（负=高估）为准。
- **自证回测（实测·50 案）**：`analysis/calib_backtest.py` 用 per_case.json 50 案算 Bias = mean(realized−pred) = **−1.581**（mean_pred 1.810 / mean_real 0.229），−Bias=1.581 ✓ 复现 metrics.json 的 +1.58 —— 证 eval-harness 点对账逻辑正确。
- **价值/边界**：即时价值=把 v1.9 一次性硬编码偏置升级为每轮自动产出的监测量 + 满 ≥5 run 据 Bias 收窄画样预期提分上界；边界=门① 真实 run 现状=0，第二层区间回调"写进去常年待触发"、诚实标未生效。
- **落地**：`tools/eval-harness.sh`（pred_gain 触发校准行·缺省跳过）+ `references/scoring.md` 硬规则5 扩回调子句 + calib 夹具入自测（4→8 夹具）。

## YZ-DET-D7 · 确定性验证器证据层（2026-06-16 · v1.10 Wave2 D7）

- **机制**：可程序判对错的脚本/工程型 skill 加 det 证据层（与盲评互补·形态分流）。scores.json `det:true`+`pass_orig/pass_cand`∈{0,1}；eval-harness det 项不进盲评聚合、单算 det_lift=Σpass_cand−Σpass_orig，**det 门=det_lift≥1 且零回归**；混合输入盲评+det 都过才通过。
- **阻断级修复（deepdive D7 预言·已实测复现并修）**：Wave1 加的"必填 prompt/orig/cand"校验会拒 det 项；旧版盲评聚合对无 orig/cand 的 det 项做除法会 null 崩。重写：校验按 det/盲评分别要求字段；盲评聚合 `select(.det!=true)` 排除 det；**空盲评集（纯 det）守卫 n>0 跳过盲评段**（防除零）。
- **自测（实测·3 夹具）**：det-pass(det_lift≥1→exit0) / det-regress(某题 pass_cand0&pass_orig1→exit1) / det-mixed(盲评+det 都过→exit0)；**9 盲评夹具零回归**（重写未破盲评路径）。eval-harness 自测 9→12 夹具。
- **形态分流 + 断言陷阱防护**：纯问诊/认知/风格型禁用强制盲评（强上退化数关键词）；剔永真断言 + 断言语义覆盖下限 + oracle 自校验（断言对参考解须 pass）。
- **待测（YZ-DET-D7 预注册）**：50 案标"可判/不可判/复合"得 p_det（预期 15-30%·窄但真）+ 可判子集 det 门方差≈0 vs 盲评 panel 方差。
- **来源**：SkillsBench(arXiv:2602.12670 确定性验证器) / SkillRevise(2606.01139 verifier-driven +25.58pp·Appendix E 断言陷阱)。

## YZ-CROSSOVER-D6 · 比样 crossover（2026-06-16 · v1.11）

- **机制**：比样多候选默认 best-of-3 独立取优；crossover 子模式（默认关）在两强候选**各擅长不同实例/维度**时生成 merge 候选 C 融合互补优点（GEPA System-Aware Merge / EvoPrompt crossover 营造化）。**严格采纳门**：① C 逐实例 Pareto 不劣于 best(A,B) + realized margin≥1.0 ② merge diff 可追溯 ③ C 新增行为审计；缺一弃 C 回退 best(A,B)（棘轮不破）。crossover 选出的候选仍走正常验证门①（crossover 在比样步内、门①下游不变）。
- **诚实预期（deepdive D6 对抗评审·本就是缓议项）**：GEPA merge 实测仅 +2%（峰值 5%）、Qwen3-8B 有负迁移；EvoPrompt crossover 平均仅 +2.5~3.5%；**单文件 skill 多无可切分互补区块（触发率预期 <10%）、严格采纳门下采纳率趋近 0** → 默认关、定位"少数真互补案例锦上添花"、**全样本期望增益≈0**。无互补区块不强行 merge（防负迁移）。
- **待测（YZ-CROSSOVER-D6 预注册）**：比样大修样本中 crossover 触发率 + 采纳率 + 采纳案例的真实 margin；预期触发<10%、采纳≈0、采纳者 margin 小。
- **定位**：纯方法论增强（crossover 是打磨 Agent 的 LLM 动作·无 harness 改动）；价值在"机制前沿对齐 GEPA/EvoPrompt"而非"普涨"。
- **来源**：GEPA(arXiv:2507.19457 Pareto+System-Aware Merge)/EvoPrompt(2309.08532 LLM-as-crossover)。
